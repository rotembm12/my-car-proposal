The Problem

The current app works, but it's showing its age. Every friction point is a lost customer.

03
My Car

Current State

What's Broken

15+ Screen Registration
3-part onboarding with info walls, separate document uploads, and an iframe payment form. Users drop off before finishing.
8-Tap Booking Flow
Location > dates > search > loading > result > pricing > confirm > done. Every extra tap loses users.
Hidden Unlock Code
Users must "slide right" to reveal the car code - a non-standard gesture that confuses first-timers standing at the car.
Dated Visual Design
Flat mint green, generic illustrations, browser-alert modals, no dark mode. Doesn't inspire trust in a premium service.
Broken Web Experience
Pricing tables, terms, and payments open in embedded web views that break the native feel. Web app can't do full booking.
No Modern Auth or Payments
No biometric login (Face ID / fingerprint). No Apple Pay or Google Pay. Payment via Tranzila iframe feels insecure.
04

The Solution

A complete rewrite. New design system. Streamlined flows. Modern stack. Full platform parity.

Everything the driver needs, visible at a glance. No hidden gestures, no buried info.

UX Architecture

Navigation: Bottom Tabs, Not Hamburger

Every primary action is one tap away. The hamburger menu is eliminated.

🗺
Map
Home screen. Find & book cars. Transforms into Active Rental dashboard when a rental is live.
📅
My Bookings
Upcoming reservations. Extend, cancel, or start early. One-tap access to scheduled rentals.
🕐
History
Past rentals with full details. Cost breakdowns, damage photos, downloadable receipts.
👤
Profile
Personal info, license, payments, credits, settings, language, support, pricing, terms.

Secondary items (pricing table, usage instructions, terms, contact us, language toggle) move into Profile > Settings. Nothing is hidden - it's just organized by frequency of use.

Accessibility & Standards

WCAG 2.1 AA
All interactive elements meet contrast ratios. Minimum 44x44pt touch targets. Support for dynamic text sizing and font scaling.
Screen Reader Support
Full VoiceOver (iOS) and TalkBack (Android) compatibility. Meaningful labels on all interactive elements.
RTL Native Support
Hebrew (RTL) as primary language. Layout flips automatically. i18next from day one with externalized strings.
Offline Resilience
Cached data shown on poor connectivity. Actions queued and synced when connection restores. Never stranded mid-rental.

Quality Assurance

Testing Strategy

Test external behavior, not implementation details. Prefer integration tests over unit tests for UI flows.

What We Test

  • Auth flow: OTP, biometrics, session, approval gating
  • Registration: full 5-step E2E, validation, persistence
  • Booking: immediate, scheduled, one-way, cancel, extend
  • Damage reporting: all 5 angles, skip prevention, upload retry
  • Payment: provider abstraction, card entry, Apple/Google Pay
  • Map: filters, view toggle, bottom sheet, location search
  • Admin: role-based access, CRUD operations, approval flow

Tools

Jest + React Native Testing Library
Component and integration tests. Mock API with MSW.
Detox / Maestro
End-to-end tests on real mobile devices and simulators.
Playwright
End-to-end tests for the web app and admin panel.

Rollout

Migration Strategy

Continuity

  • Published as update to existing app listings (preserves ratings, downloads, store presence)
  • Existing users log in with same phone number and see full history
  • Feature flag rollout - gradually enable for user segments
  • Rollback plan: keep old app version deployable during transition

Dependencies

  • UI/UX Brand Guidelines: New design system depends on guidelines currently in development. Component work can start with placeholder tokens.
  • API Documentation: Comprehensive docs needed for existing backend endpoints.
  • App Store Accounts: Existing Apple Developer and Google Play Console accounts.
